# US-004 - MegaROM Support for MSXBAS2ROM

## Story

As an MSX-BASIC developer, I want the debugger to support MegaROM builds produced by MSXBAS2ROM so that I can debug larger programs that require bank switching.

## Acceptance Criteria

- MegaROM images can be launched by the VSCode debug session using openMSX.
- The debugger can attach to a running MegaROM program.
- Source-level debugging remains functional when MegaROM bank switching occurs.

## BDD Scenarios

### Scenario 1: Start a debug session with a MegaROM image

Given I have a MegaROM image generated by MSXBAS2ROM
When I start a debug session in VSCode
Then openMSX launches with the MegaROM image loaded
And the program starts under debugger control

### Scenario 2: Breakpoints work in MegaROM programs

Given a MegaROM program is running under the debugger
And I set a breakpoint on a source line
When I continue execution
Then execution stops at the breakpoint line in VSCode

### Scenario 3: Debugging stays accurate after bank switching

Given a MegaROM program performs bank switching
When I step over or continue execution
Then the active source line remains correct in VSCode
And breakpoints in the current bank continue to function

## Technical specification

Use the following guidelines:

- Current active segment is found at 0xC023 (8 bits unsigned), read it with `peek 0xC023`;
- MSX-BASIC GOSUBs always push 3 integers to the Z80 call stack (SP): callback megaROM helper address, followed by segment and address to the next MSX-BASIC line to return. You can read it using `peek_u16 [reg SP]`.
