# US-003.1 - Continue Execution

## Story

As an MSX-BASIC developer, I want to continue execution so the program runs without stopping on automatic breakpoints.

## Acceptance Criteria

- Continue runs the program and skips automatic breakpoints.
- Execution stops only on manual breakpoints and the end-of-program breakpoint.

## BDD Scenarios

### Scenario 1: Continue from a paused state

Given the program is paused
When the user executes Continue (F5)
Then automatic breakpoints must be disabled
And manual breakpoints must remain enabled
And the end-of-program breakpoint must remain enabled
And the emulator must continue execution (debug cont)

### Scenario 2: Continue with no manual breakpoints

Given the program is paused
And there are no manual breakpoints defined
When the user executes Continue (F5)
Then automatic breakpoints must be disabled
And the end-of-program breakpoint must remain enabled
And the emulator must continue execution (debug cont)
And execution must only stop at the end-of-program breakpoint

## Technical specification

Use the following guidelines:

- Continue disables all automatic breakpoints (LIN\_\*), while keeping manual breakpoints and end-program breakpoint active.

Procedure when continuing:

1. Disable automatic breakpoints.
2. Ensure manual breakpoints and end-program breakpoint are enabled.
3. Continue emulation: `debug cont`.
