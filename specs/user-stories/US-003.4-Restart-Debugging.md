# US-003.4 - Restart Debugging

## Story

As an MSX-BASIC developer, I want to restart debugging after the program ends so I can run it again quickly.

## Acceptance Criteria

- When the program ends, the user is prompted to restart.
- If the user confirms, the current debug session stops and a new one starts with the same configuration.
- If the user declines, debugging stops without restarting.

## BDD Scenarios

### Scenario 1: User accepts restart prompt

Given the program reaches its end
And the debugger emits an end-program notification
When the user confirms restart
Then the current debug session must stop
And a new debug session must start with the same configuration

### Scenario 2: User declines restart prompt

Given the program reaches its end
And the debugger emits an end-program notification
When the user declines restart
Then the current debug session must stop
And no new debug session should start

## Technical specification

Use the following guidelines:

- Restart is triggered by the end-program event.
- The prompt is modal and blocks until the user chooses.

Procedure when restarting:

1. When receiving the end-program event, display a modal prompt asking to restart.
2. If the user accepts:
   - Stop the current debug session.
   - Start a new debug session with the same workspace folder and configuration.
3. If the user declines:
   - Stop the current debug session only.
