# US-003.3 - Stack Trace: Step Over

## Story

As an MSX-BASIC developer, I want to execute the current line and moves to the next line in the same scope so I can treat function calls as a single step without entering them.

## Acceptance Criteria

- Stepping Over works as expected.

## BDD Scenarios

### Scenario 1: Step Over on a simple line (no function call)

Given the program is paused at a line that does not contain a function call
And the current stack pointer is stored (reg SP)
When the user executes Step Over
Then a Step Into must be executed
And the new stack pointer must be retrieved (reg SP)
And the new stack pointer must be equal to the previous one
And no Step Out must be executed
And execution must pause at the next line in the same scope

### Scenario 2: Step Over on a function call

Given the program is paused at a line that contains a function call
And the current stack pointer is stored (reg SP)
When the user executes Step Over
Then a Step Into must be executed
And the new stack pointer must be retrieved (reg SP)
And the new stack pointer must indicate a deeper stack (lower value)
And a Step Out must be executed
And execution must pause at the next line after the function call
And the call stack must return to the original scope

### Scenario 3: Step Over at root scope with no function call

Given the program is paused at the root scope
And the current line does not contain a function call
When the user executes Step Over
Then a Step Into must be executed
And the stack pointer must remain unchanged
And no Step Out must be executed
And execution must pause at the next line

### Scenario 4: Step Over on nested function calls

Given the program is paused at a line containing nested function calls
And the current stack pointer is stored (reg SP)
When the user executes Step Over
Then a Step Into must be executed
And the stack pointer must decrease indicating entry into a deeper scope
And a Step Out must be executed
And execution must return to the original stack level
And execution must pause at the next line in the same scope

### Scenario 5: Step Over preserves breakpoint behavior

Given the program is paused
And there are active breakpoints in the program
When the user executes Step Over
Then the internal Step Into and possible Step Out operations must follow their respective breakpoint handling rules
And execution must pause at the correct next line in the same scope
And no unintended breakpoint must alter the expected Step Over behavior

### Scenario 6: Step Over with unchanged stack after Step Into

Given the program is paused
And the current stack pointer is stored (reg SP)
When the user executes Step Over
And the stack pointer after Step Into remains unchanged
Then no Step Out must be executed
And execution must proceed normally to the next line

## Technical specification

Use the following guidelines:

- Stepping over means keeping in the current scope of the call stack.

Procedure when stepping over:

1. Save the current stack scope: `reg SP`;
2. Execute a Step Into (US-003.1);
3. Get the new current stack scope: `reg SP`;
4. If the new stack scope is below the last one, execute a Step Out (US-003.2).
5. Otherwise, do nothing.
