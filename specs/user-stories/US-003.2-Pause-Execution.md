# US-003.2 - Pause Execution

## Story

As an MSX-BASIC developer, I want to pause execution so the program stops at the next automatic breakpoint.

## Acceptance Criteria

- Pause arms automatic breakpoints and resumes execution until the next stop.

## BDD Scenarios

### Scenario 1: Pause while program is running

Given the program is running
When the user executes Pause
Then automatic breakpoints must be enabled
And the emulator must continue execution (debug cont)
And execution must pause at the next automatic breakpoint or manual breakpoint

### Scenario 2: Pause when already paused

Given the program is paused
When the user executes Pause
Then automatic breakpoints must be enabled
And the emulator must continue execution (debug cont)
And execution must pause at the next automatic breakpoint or manual breakpoint

## Technical specification

Use the following guidelines:

- Pause enables automatic breakpoints and continues emulation to stop at the next line breakpoint.

Procedure when pausing:

1. Enable automatic breakpoints (LIN\_\*).
2. Continue emulation: `debug cont`.
